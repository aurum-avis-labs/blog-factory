Der Anfang ist der schwierigste Teil. Eine leere Leinwand kann Systemdenken abstrakt erscheinen lassen, selbst wenn Ihr Problem schmerzlich real ist — ob es sich um eine geschäftliche Herausforderung, ein politisches Rätsel, eine investigative Frage oder eine Advocacy-Kampagne handelt. Dieses Holist-IQ Tutorial gibt Ihnen einen einfachen, wiederholbaren Weg, um Ihre erste Feedbackschleifen-Karte zu erstellen und sie in testbare Aktionen umzuwandeln.

Sie müssen nicht alles modellieren. Sie müssen nur genug modellieren, um die Schleifen zu sehen.

## Holist-IQ Tutorial: Was Sie erstellen werden

Am Ende dieses Holist-IQ Tutorials haben Sie:
- Ein kleines Feedbackschleifen-Diagramm (10–15 Variablen)
- Mindestens eine verstärkende Schleife und eine ausgleichende Schleife
- Einen Hebelpunkt zum Testen
- Eine kurze Liste von Annahmen zur Validierung

Wählen Sie ein reales Problem. Das Tutorial funktioniert am besten, wenn das Ergebnis wichtig ist.

## Schritt 1: Wählen Sie ein messbares Ergebnis

Schreiben Sie einen Satz:
- "Wir wollen die Kundenabwanderung in den ersten 30 Tagen reduzieren." (Wirtschaft)
- "Wir wollen verstehen, warum diese Regulierung immer wieder umgangen wird." (Politik)
- "Wir wollen nachvollziehen, warum dieses systemische Problem trotz Reformen immer wieder auftaucht." (Journalismus)
- "Wir wollen herausfinden, warum unsere Advocacy-Erfolge mit der Zeit erodieren." (Lobbying)

Fügen Sie eine grobe Kennzahl hinzu, wenn möglich:
- Abwanderungsrate, Rückstandsgröße, Durchlaufzeit, NPS-Trend, Fehlerquote

Das hält Ihr Causal-Loop-Diagramm geerdet.

## Schritt 2: Listen Sie die Variablen auf, die das Ergebnis beeinflussen

Brainstormen Sie Treiber, ohne sie noch zu debattieren. Beispiele für Abwanderung:
- Onboarding-Abschlussrate
- Time-to-Value
- Produktkomplexität
- Support-Reaktionszeit
- Erwartungsabweichung
- Wahrgenommener ROI
- Kundenvertrauen

Zielen Sie auf 10–15 Variablen. Zu wenige und Sie verpassen Dynamiken. Zu viele und Sie verlieren Klarheit.

## Schritt 3: Verbinden Sie Variablen mit Ursache-Wirkungs-Verbindungen

Für jede Verbindung fragen Sie:
- "Wenn dies zunimmt, nimmt das (letztendlich) zu oder ab?"

Schreiben Sie es zuerst in einfacher Sprache:
- "Höhere Produktkomplexität erhöht Support-Tickets."
- "Längere Support-Reaktionszeit erhöht Frustration."
- "Höhere Frustration erhöht Abwanderung."

Wenn der Effekt später auftritt, markieren Sie ihn als Verzögerung. Verzögerungen sind dort, wo Überraschungen leben.

## Schritt 4: Finden Sie Ihre erste Feedbackschleife

Suchen Sie nach einer Kette, die zum Ausgangspunkt zurückkehrt.

**Ein Beispiel für eine verstärkende Schleife:**
- Komplexität nimmt zu → Tickets nehmen zu → Reaktionszeiten verschlechtern sich → Frustration nimmt zu → Abwanderung nimmt zu → Druck, mehr Features zu liefern, nimmt zu → Komplexität nimmt zu

**Ein Beispiel für eine ausgleichende Schleife:**
- Tickets nehmen zu → Einstellungen nehmen zu → Kapazität steigt → Reaktionszeiten verbessern sich → Frustration nimmt ab → Tickets stabilisieren sich

Benennen Sie die Schleife so, dass Ihr Team sie sich merkt:
- "Support-Spirale"
- "Einstellungs-Aufholschleife"
- "Feature-Druck-Schleife"

Benennung hilft bei der Stakeholder-Abstimmung. Für einen tieferen Einblick, wie diese Schleifen funktionieren, lesen Sie über [Systemdenken im Unternehmen](/de/blog/systemdenken-im-unternehmen).

## Schritt 5: Identifizieren Sie einen Hebelpunkt

Fragen Sie:
- "Wo könnte eine kleine Änderung eine bedeutende Verschiebung erzeugen?"

Häufige Hebelpunkte:
- Verzögerungen reduzieren (schnelleres Feedback, frühere Erkennung)
- Informationsfluss verbessern (Sichtbarkeit, gemeinsamer Kontext)
- Anreize ändern (was belohnt wird, wird wiederholt)
- Reibung entfernen (Übergaben, Genehmigungen, unklare Kriterien)

Wählen Sie einen Hebelpunkt, den Sie in den nächsten zwei Wochen tatsächlich beeinflussen können.

## Schritt 6: Verwandeln Sie die Karte in einen testbaren Aktionsplan

Schreiben Sie ein Experiment in diesem Format:
- **Wenn** wir X ändern (Intervention),
- **erwarten wir**, dass Y (Frühindikator) sich innerhalb von Z bewegt,
- **und wir beobachten** Nebeneffekt W.

Beispiel:
- Wenn wir die Onboarding-Schritte für die Top-2-Kundensegmente vereinfachen,
- erwarten wir, dass die Onboarding-Abschlussrate innerhalb von 14 Tagen steigt,
- und wir beobachten Support-Tickets bezüglich Verwirrung über neue Kommunikation.

So wird Entscheidungskartierung zu echter Arbeit -- datengetriebene Entscheidungsfindung, die tiefer geht als das, was Dashboards allein aufdecken können.

## Schritt 7: Erfassen Sie Annahmen, die Sie validieren müssen

Wählen Sie 3–5 Verbindungen, die sich unsicher anfühlen, und schreiben Sie auf, was sie widerlegen würde.

Beispiele:
- "Wir nehmen an, dass Reaktionszeit mehr Abwanderung verursacht als Produktmängel."
- "Wir nehmen an, dass Rabatte das Abwanderungsrisiko durch weniger passende Kunden erhöhen."
- "Wir nehmen an, dass Komplexität schneller wächst als unsere Fähigkeit, sie zu unterstützen."

Das hält Ihr Modell ehrlich und verhindert übermäßiges Vertrauen.

## Häufige Fehler (und schnelle Lösungen)

**Versuchen, alles zu modellieren.**
Lösung: Halten Sie es klein. Sie können später erweitern.

**Zu früh über "die Wahrheit" streiten.**
Lösung: Erfassen Sie mehrere Hypothesen als alternative Verbindungen. Testen Sie später.

**Verzögerungen überspringen.**
Lösung: Markieren Sie Verzögerungen explizit. Sie erklären die meisten Überraschungen.

**Die Karte im Workshop lassen.**
Lösung: Planen Sie eine kurze Überprüfung nach Ihrem Experiment. Aktualisieren Sie das Modell.

## Den Kreislauf schliessen

Ein Feedbackschleifen-Diagramm ist nur nützlich, wenn es sich mit der Realität weiterentwickelt. Nutzen Sie dieses Holist-IQ Tutorial einmal, dann wiederholen Sie es, wann immer ein Problem zurückkehrt.
